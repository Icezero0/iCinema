# 2 产品概述

iCinema 是一个支持多人在同一房间中协同观看视频内容的系统。

用户可以创建房间或加入已有房间，在房间中加载视频资源并进行同步播放，同时通过消息进行交流。

系统核心能力包括：

- 用户系统
- 房间系统
- 房间成员管理
- 房间消息通信
- 视频同步播放

---

# 3 系统总体架构

系统采用前后端分离架构。

前端通过 HTTP API 与后端进行业务通信，通过 WebSocket 建立实时连接以接收房间内事件同步。

系统结构：

Frontend  
→ HTTP API → Backend  
→ WebSocket → Realtime Service  

Backend 负责业务逻辑与数据管理，同时承担 WebSocket 实时通信能力。

---

# 4 前端系统结构（总览）

前端负责界面呈现、用户交互、播放器控制以及实时状态响应。

主要模块包括：

- 用户模块
- 房间模块
- 播放器模块
- 聊天模块
- WebSocket 状态管理模块
- 全局 UI 与布局模块

前端通过 HTTP 获取系统状态，并通过 WebSocket 接收实时事件。

---

# 5 后端系统结构（总览）

后端采用模块化单体架构，并按照业务领域划分模块。

主要模块包括：

- auth
- users
- rooms
- messages
- realtime

每个模块内部采用分层结构组织：

- router
- service
- repository
- models
- schemas

该结构用于分离接口层、业务逻辑层与数据访问层。

---

# 6 数据通信方式

系统存在两类主要通信方式。

## 6.1 HTTP API

HTTP API 用于处理需要持久化或可查询的业务操作，例如：

- 用户登录
- 用户信息获取与修改
- 房间创建与查询
- 消息历史查询

## 6.2 WebSocket

WebSocket 用于房间内部的实时通信，包括：

- 实时消息广播
- 房间成员状态变化
- 播放状态同步
- 房间事件通知

# 7 核心领域设计

## 7.1 用户系统

User 表示系统用户，用于身份认证与用户资料管理。

用户拥有基础资料信息，例如用户名、头像等。

## 7.2 房间系统

Room 表示一个观影房间，是多人协作的主要空间。

房间负责承载成员关系、消息通信以及播放状态。

## 7.3 成员关系

RoomMember 表示用户与房间之间的成员关系。

该关系用于描述：

- 用户属于哪个房间
- 用户在房间中的角色

## 7.4 消息系统

Message 表示房间内的聊天消息。

消息用于房间成员之间的文本通信，并按房间进行组织。

## 7.5 播放系统

播放系统负责房间中的视频播放与同步状态维护。

### 视频来源

房间中的视频播放来源分为两类：

**网络视频资源**

用户可以提供可访问的视频资源地址，例如：

- mp4
- m3u8
- 其他浏览器可直接播放的视频 URL

**本地文件**

用户可以选择本地文件作为视频来源。

前端负责本地文件选择与播放器接入，后端负责房间状态与同步逻辑。

### 播放状态

PlaybackState 表示房间当前的视频播放状态，包括：

- 当前播放源
- 播放位置
- 播放或暂停状态

---

# 8 核心交互流程

系统主要使用流程如下：

1 用户注册或登录系统  
2 用户创建房间或加入已有房间  
3 房间内加载视频资源  
4 房间成员围绕同一播放状态进行同步观看  
5 房间成员通过消息进行交流  
6 系统通过 HTTP 与 WebSocket 协同维护房间状态

---

# 9 系统设计原则

系统设计遵循以下原则：

1 以房间为核心组织系统结构  
2 HTTP 负责持久化业务操作  
3 WebSocket 负责实时事件同步  
4 前后端职责边界清晰  
5 模块按业务领域进行划分
